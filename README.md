# CHANTHABURI FIREWORKS — เว็บไซต์ขายอุปกรณ์จุดพลุ

เว็บไซต์ static HTML/CSS/JS ล้วน ไม่ต้องมี backend หรือ build step — เปิดได้ทันที และอัปขึ้น GitHub Pages ได้เลย

## โครงสร้างไฟล์
```
site/
├── index.html          หน้าแรก
├── products.html        รายการสินค้า + ตัวกรอง
├── product.html          รายละเอียดสินค้า 1 ชิ้น (ใช้ ?id=IGN-24X)
├── portfolio.html        ผลงาน (รูป/วิดีโอ) แบ่งตามแท็บ
├── blog.html             รายการบทความ
├── post.html              บทความ 1 ชิ้น (ใช้ ?id=b1)
├── about.html
├── contact.html
├── css/style.css         ธีมทั้งเว็บ (สีดำ/น้ำเงิน โทนอุปกรณ์เทคนิค)
├── js/main.js            เมนู, footer, ฟังก์ชันช่วยเหลือ (ใช้ทุกหน้า)
└── data/data.js          ★ ข้อมูลสินค้า/ผลงาน/บทความทั้งหมดอยู่ที่นี่
```

## แก้ไขข้อมูล (สำคัญที่สุด)
**แก้ไฟล์เดียวคือ `data/data.js`** ทุกหน้าจะดึงข้อมูลจากไฟล์นี้อัตโนมัติ ไม่ต้องแตะ html เลย:

- เพิ่ม/ลบสินค้า → แก้ array `PRODUCTS`
- เปลี่ยนสถานะสต็อก → แก้ `stock: "in" | "low" | "out"`
- เพิ่มรูปสินค้า (แนะนำ 3-5 รูป) → เติมใน array `images: [...]`
- เพิ่มผลงาน → แก้ array `PORTFOLIO` (ใส่ `video: "path/to/video.mp4"` ได้ถ้ามีวิดีโอ)
- เพิ่มบทความ → แก้ array `BLOG`

ตอนนี้รูปภาพเป็น **placeholder SVG** (กรอบตารางสีน้ำเงิน/ส้มมีตัวหนังสือ) ไว้ให้เห็นโครงก่อน
ให้แทนที่ด้วยรูปจริงโดย:
1. สร้างโฟลเดอร์ `images/products/` แล้ววางไฟล์ เช่น `IGN-24X-1.jpg`
2. แก้บรรทัด `images: [placeholder(...), ...]` เป็น `images: ["images/products/IGN-24X-1.jpg", "images/products/IGN-24X-2.jpg", ...]`

## ทดสอบก่อนอัปขึ้น GitHub
เว็บนี้ใช้ JavaScript render ข้อมูล ถ้าเปิดไฟล์ index.html ตรงๆ (double-click) จะทำงานได้ปกติ
เพราะข้อมูลอยู่ใน `data.js` ไม่ใช่ JSON แยก (ไม่ติดปัญหา CORS ตอนเปิดจากเครื่อง)

ถ้าอยากรันแบบเซิร์ฟเวอร์จริง (แนะนำ เพื่อให้เหมือนตอนขึ้นจริงที่สุด):
```
cd site
python -m http.server 8000
```
แล้วเปิด http://localhost:8000

## อัปขึ้น GitHub Pages
1. สร้าง repo ใหม่บน GitHub แล้วอัปโหลดไฟล์ทั้งหมดในโฟลเดอร์ `site/`
2. ไปที่ Settings → Pages → Source เลือก branch `main` และโฟลเดอร์ `/ (root)`
3. รอสักครู่ จะได้ลิงก์ `https://<username>.github.io/<repo>/`

## ถ้าอยากให้ฟอร์มติดต่อ (contact.html) ส่งอีเมลได้จริง
เว็บ static ส่งฟอร์มเองไม่ได้ ต้องพึ่งบริการฟรีภายนอก เช่น:
- **Formspree** (formspree.io) — ง่ายที่สุด แค่เปลี่ยน `<form>` ให้ action ไปที่ endpoint ที่เขาให้
- **Getform** หรือ **EmailJS** — ทางเลือกอื่นที่คล้ายกัน

## เรื่องการเก็บข้อมูล — ควรเก็บที่ไหนดี?
ตอนนี้ใช้ `data/data.js` ซึ่ง**เหมาะสำหรับเริ่มต้น**ที่สุด เพราะ:
- ไม่ต้องมี backend/database, host ฟรีบน GitHub Pages ได้เลย
- แก้ไฟล์เดียวจบ, แก้ผ่านหน้าเว็บ GitHub ได้โดยไม่ต้องมีเครื่องมือพิเศษ

**เมื่อไหร่ควรอัปเกรด:**
- ถ้าอยากให้คนที่ไม่ใช่โปรแกรมเมอร์แก้ราคา/สต็อกได้เอง → ใช้ **Google Sheet** แล้วดึงข้อมูลผ่าน API (เช่น SheetDB, sheet.best) แทนไฟล์ data.js
- ถ้าต้องการระบบตะกร้า/สั่งซื้อ/ชำระเงินจริง → ต้องมี backend เช่น Shopify, หรือใช้ Firebase/Supabase เป็นฐานข้อมูล + ระบบชำระเงินแยก (Omise, 2C2P)
- ถ้าสินค้าจะเปลี่ยนบ่อยมากและมีทีมหลายคนแก้ไข → พิจารณาใช้ headless CMS ฟรี เช่น **Decap CMS** ซึ่งแก้ผ่านหน้าเว็บสวยๆ แต่ยังเก็บข้อมูลเป็นไฟล์ใน GitHub repo เหมือนเดิม (ไม่ต้องมี server เพิ่ม)

สรุป: เริ่มจาก `data.js` ไปก่อน ใช้งานจริงแล้วค่อยดูว่าจำเป็นต้องอัปเกรดหรือไม่
